# 前端規劃

React + Vite + TypeScript,**表單式查詢先行**;聊天機器人列為後續 overlay(另一個前門,共用同一組後端)。

## 頁面

1. **首頁(已實作)**:鄉鎮下拉 + 日期選擇 → 天氣卡片 + 降雨機率視覺化 + AI 行前建議區塊。
2. **行前規劃頁(P2)**:天氣 + 景點資訊卡。
3. **系統說明頁**:架構、技術棧、API/雲端流程概覽(給評審看設計思路)。

## 前後端邊界

- 前端只存 UI state,不持有商業邏輯。
- 風險分級、資料正規化、AI 摘要全在後端。
- 第三方 API 僅由後端存取,金鑰不進前端 bundle。

## 已實作元件

- `TripForm`:鄉鎮下拉 + 日期(限未來 7 日內)+ 送出。
- `ForecastView`:AI 摘要卡 + 每日預報卡(高低溫、降雨機率長條、建議)。
- `lib/api.ts`:呼叫後端;後端不可達時有 mock fallback,前端可獨立 demo。

## 串接方式

- Dev:Vite 代理 `/api` → `localhost:8080`,免 CORS 煩惱。
- Prod:`VITE_API_BASE` 指向 Cloud Run URL;後端 CORS 白名單放前端網域。

## 後續(景點粒度)

TDX 景點 API 為**縣市層級**,但使用者選鄉鎮:先做 `鄉鎮 → 縣市` 正規化,再用鄉鎮中心座標 + 半徑過濾;對外用語為「**目的地周邊景點**」,不承諾「鄉鎮專屬景點」。
